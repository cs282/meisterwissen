Projekt Meisterwissen. Zweck: Erfahrungswissen des Betriebs erfassen (Baustellen-Aufnahmen → Interview-Bot "Werkstatt-Willi" → strukturierte Wissensbausteine), später per RAG abrufbar. Fokus: NUR Innenanstriche. Taxonomie: A1 Gipsputz/Kalk-Zement-Putz, A2 Gipskarton Q2–Q4, A3 Altanstrich tragfähig/nicht tragfähig, A4 Tapete als Untergrund, A5 Beton/Sichtbeton, A6 Sonderfälle Nikotin/Wasserflecken/Schimmel/Leimfarbe. B1 Dispersion matt (Brillux Superlux 3000, Caparol Indeko-plus), B2 Latex/abwaschbar, B3 Silikat/mineralisch, B4 Kalkfarbe, B5 Isolier-/Sperrgründe, B6 Grundierungen (Tiefgrund/Haftgrund/Aufbrennsperre). C1 Rollen Florhöhen 12/18/21mm, C2 Pinsel, C3 Airless, C4 Abklebetechnik, C5 Hilfsmittel. D1 Nass-in-nass, D2 Vorlegen+Nachrollen, D3 Zwischen-/Schlussbeschichtung Trocknungszeiten, D4 Farblich abgesetzte Flächen, D5 Raumklima. E1 Streifen/Ansätze, E2 Wolkenbildung, E3 Abplatzungen/Blasen, E4 Durchschlagende Flecken, E5 Glanzgrad – bei E immer Ursache→Diagnose→Lösung. Stack: Next.js 15 App Router TypeScript, Supabase (Postgres+Storage+pgvector), Whisper API (deutsch), Claude API claude-sonnet-4-6. Statusflow: draft → interviewed → reviewed → published, kein published ohne Meister-Freigabe. UI: Deutsch, mobile-first. Regel: Interview-Bot erfindet NIEMALS Fachwissen. Arbeitsweise: Es wird strikt Baustein für Baustein gebaut, niemals vorgreifen.
